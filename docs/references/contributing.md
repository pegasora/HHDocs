# Contributing Guide

> **How to contribute to Jarvis development**

## Ways to Contribute

1. **Report Bugs** - Use GitHub Issues
2. **Suggest Features** - Open a discussion
3. **Improve Documentation** - Submit PRs to docs
4. **Write Code** - Follow contribution guidelines

## Development Process

### 1. Fork & Clone
```bash
git clone https://github.com/pegasora/local-ai.git
cd local-ai
```

### 2. Create Branch
```bash
git checkout -b feature/my-feature
```

### 3. Make Changes
- Follow code style guidelines
- Write tests for new features
- Update documentation

### 4. Submit PR
- Write clear commit messages
- Reference related issues
- Request review from maintainers

## Code Style

- **TypeScript**: Strict mode, proper types
- **Formatting**: Prettier (run `npm run prettier:fix`)
- **Linting**: ESLint (run `npm run lint`)
- **Naming**: camelCase for variables, PascalCase for components
