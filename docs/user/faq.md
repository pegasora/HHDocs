# Frequently Asked Questions

> **Common questions about using Jarvis**

## General Questions

**Q: What is Jarvis?**  
A: Jarvis is H&H's self-hosted AI assistant for injection molding operations. It runs entirely on local infrastructure.

**Q: Is my data safe?**  
A: Yes! All data stays local. Nothing is sent to external servers.

**Q: Can I access Jarvis from home?**  
A: Only via Tailscale VPN if configured. Contact IT for remote access.

## Using Jarvis

**Q: How do I ask questions?**  
A: Just type naturally, like you're asking a coworker.

**Q: What kind of questions can I ask?**  
A: Anything about injection molding, H&H procedures, defect troubleshooting, machine operation, etc.

**Q: How do I cite Jarvis in reports?**  
A: "According to Jarvis AI (source: [document-name.pdf])"

## Technical Questions

**Q: Which documents does Jarvis know about?**  
A: Ask "List all available documents" to see the complete list.

**Q: How often is the knowledge base updated?**  
A: Documents are ingested automatically when added to /tank/local-ai/data

**Q: Can Jarvis access live machine data?**  
A: Not yet. Press20 integration is planned for Q1 2026.

## Troubleshooting

**Q: Jarvis won't load**  
A: 1) Check WiFi connection, 2) Try different browser, 3) Contact IT

**Q: My chats disappeared**  
A: Check if you're logged in as the correct user. Chats are per-user.

**Q: Jarvis is slow**  
A: May be under heavy load. Try refreshing or wait a moment.
